Lesson 1 - Modules

  • An AngularJs Module defines as application.
  • The Module is a container for the different parts of an application.
  • The Module is a container for the application controllers.
  • Controllers always belong to a Module.

Creating a Module

A Module is created by using AngularJs function angular.module 
			<div ng-app="myApp">
				...
			</div>
			<script>
				var app = angular.module('myApp',[]); 
			>/script>
  • The "myApp" parameter refers to an HTML element which the application will to run
  • Now, you can add controllers, directives, filters and more, to your AngularJs aplication

Adding a controller

Add a controller in your application and refer to controller with ng-controller directive 
			<div ng-app="myApp" ng-controller="myController">
				{{firstName + ' ' + lastName}}
			</div>
			<script>
				var app = angular.module('myApp',[]);
				app.controller('myController',function($scope){
					$scope.firstName = 'dat';
					$scope.lastName = 'feed';
				}); 
			</script>

Adding directive

  • AngularJs has a set of built-in directives which you can add functionality to your application
  • In addition you can use the module to add your own directives to your application 
			<div ng-app="myApp" datfeed-directive>

			</div>
			<script>
				var app = angular.module('myApp',[]);
				app.directive('datfeedDirective',function(){
					return{
						template: 'I am Datfeed, I am leaning Angular Js, I was made in a directive contructor'
					}
				}); 
			</script> -->

Modules and Controllers in Files

  • It is common in AngularJs applications to put the module and controllers in Javascript files
  • In this example, "myApp.js" contains a module defination, while "myController.js" contains the controller
{{firstName + " " + lastName}}

  • myApp.js

    					var app = angular.module('myApp',[]);
    • The [] parameter in the module definition can be used to define dependent modules
    • Without the [] parameter, you are not creating a new module, but retrieving a existing one.

  • myController.js

    					app.controller('myController',function($scope){
						$scope.firstName = 'dat';
						$scope.firstName = 'feed';
					});

When to Load the Library

  • While it is common in HTML applications to place scripts at the end of 'body' element. It is recommended that you load the AngularJs Library either in the 'head' or at the start of the 'body'
  • This is because calls to angular.module can only be complied after the library has been loaded
